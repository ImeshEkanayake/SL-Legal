# V2 Phase 10 Release Asset Publication Contract

## Scope

Phase 10 controls how approved evidence bundles are planned for GitHub release asset publication.

No V1 files, raw corpus data, release logs, database schema, or normal Git contents are changed.

## Canonical Manifest

The canonical publication manifest is:

```text
rag/evals/phase10_release_asset_publication.json
```

It defines:

- GitHub repo
- target release tag
- approved asset paths
- labels
- required status
- content type

Only paths under:

```text
logs/release-artifacts/
```

are eligible for publication. The planner blocks `data`, `.env`, `.git`, `node_modules`, and `.next` paths.

## Plan Mode

Plan publication without mutating GitHub:

```bash
PYTHONPATH=rag python3 scripts/publish_phase10_release_assets.py \
  --output logs/release-artifacts/phase10-publication-plan.json
```

Detached plan:

```bash
scripts/run_detached_quality_gate.sh asset-publication-plan phase10-asset-publication-plan
```

## Execute Mode

Upload approved assets only after reviewing the plan:

```bash
PYTHONPATH=rag python3 scripts/publish_phase10_release_assets.py \
  --execute \
  --clobber \
  --output logs/release-artifacts/phase10-publication-result.json
```

Execution runs:

```text
gh release upload <tag> <asset> --repo <repo>
```

## Required Local Assets

Phase 10 expects:

- `logs/release-artifacts/phase9-release-evidence.tar.gz`
- `logs/release-artifacts/phase9-artifact-report.json`

These are generated by:

```bash
scripts/run_detached_quality_gate.sh artifact-report phase9-artifact-report
```

## Safety Boundaries

- Publication plan must be reviewed before `--execute`.
- Do not publish raw corpus data.
- Do not publish `.env` files.
- Do not publish `node_modules`, `.next`, or `.git` contents.
- Do not commit generated release assets to normal Git.
